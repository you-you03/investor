## 01. プロジェクト概要

### ビジョン
LLMを中核に据えた自律型の米国株投資エージェントを構築する。MVPフェーズでは「AIが調査・分析・提案を行い、人間が執行判断を承認する」ヒューマン・イン・ザ・ループ型の設計とし、将来的には取引APIとの連携による完全自動化を目指す。

### 投資方針
- 対象：米国株（1株単位、個人でも参入しやすい銘柄中心）
- スタイル：積極的（高リスク・高リターン志向）
- ホライズン：中長期（1週間〜数ヶ月）
- スクリーニング：AIが自律的に候補を選定（条件は固定しない）

## 02. 3エージェント構成

### 🔍 Research Agent（リサーチエージェント）
- 市場全体をスキャン
- 財務データ収集・分析
- ニュース・SNSトレンド取得
- テクニカル指標算出
- アナリストレポート参照
- 候補銘柄リスト生成

### 🧠 Decision Agent（意思決定エージェント）
- リサーチ結果を受け取り統合
- 買い・売り・ホールド判断
- 期待値スコア・推奨度算出
- 投資レポート生成
- Slack通知で人間に提示

### 📡 Monitor Agent（モニタリングエージェント）
- 保有銘柄を日次監視
- 値動きウォッチ
- 関連ニュース継続取得
- 異常検知でDecision Agentへ連携
- デイリーサマリー生成

### MVPワークフロー
1. リサーチトリガー
2. 銘柄スキャン
3. 分析レポート生成
4. Slack通知・提案
5. 人間が承認・執行
6. 日次モニタリング
7. 売り提案 → 人間執行

## 04. 技術スタック推奨

### LLM / Agent
- 推奨: Claude (Anthropic)
- 理由: ツール連携・エージェント構築に最も優れ、Claude.aiのArtifacts上でそのまま構築・検証可能。将来のMCP連携も容易。
- 判定: → 推奨

### 株価・財務データAPI
- 推奨: Polygon.io
- 理由: 無料枠あり、米国株データ充実、RESTful APIが使いやすくMVP向き。財務データはFinancial Modeling Prepと併用も可。
- 判定: → 推奨

### ニュース・トレンド
- 推奨: NewsAPI / Perplexity
- 理由: NewsAPIでニュース取得、PerplexityのSonar APIでリアルタイムWeb検索を組み合わせると精度が高い。
- 判定: → 推奨

### 通知
- 推奨: Slack API (Webhook)
- 理由: 既に要件確定済み。Incoming Webhookで最短実装可能。
- 判定: ✓ 確定

### 証券口座 (将来)
- 推奨: Alpaca
- 理由: 米国株対応、無料ペーパートレード環境あり、REST/WebSocket APIが充実。MVP後の自動化フェーズに最有力。
- 判定: → 推奨（将来）

### フロントエンド / UI
- 推奨: Next.js + Tailwind
- 理由: ダッシュボード形式のUIをすばやく構築できる。Claude Artifactsでプロトタイプ → 本実装の流れが効率的。
- 判定: → 推奨

## 05. MVPロードマップ（4週間）

### Week 1: 基盤構築フェーズ
- 技術スタック確定（LLM・データAPI選定）

### Week 2: コア機能実装フェーズ

### Week 3: モニタリング・UI構築フェーズ
- Monitor Agent実装（日次監視・売り提案）
- ダッシュボードUI構築

### Week 4: 検証・調整フェーズ
- 実データでのエンドツーエンドテスト

- MVP完成・デプロイ
- 次フェーズ（自動取引連携）の設計開始

## 06. リスクと対策

### HIGH RISK: AIの判断ミスによる損失
MVPは必ず人間が最終承認。自動執行はペーパートレード（仮想）で十分に検証してから移行する。

### HIGH RISK: データAPIのレート制限・コスト
MVP段階は無料枠で検証。Polygon.ioの無料プランで開始し、必要に応じて有料プランへ移行。

### MID RISK: ハルシネーション（AIの誤情報）
財務数値は必ずAPIからの実データを使用。LLMには解釈・判断のみを担わせ、数値生成はさせない設計にする。

### MID RISK: 4週間でのスコープ超過
Week 1〜2のコア機能（リサーチ・提案・Slack通知）を最優先。UIとシミュレーションはWeek 3以降に後回し可。
