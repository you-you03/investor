# investor エージェント 全体像

investor は「毎日の米国株投資判断をAIで自動化する」システムです。
大きく **4つのコマンド（ワークフロー）** が順番に連携して動きます。

---

## 全体の流れ（1日の使い方）

```
/research → /decision → /monitor（毎日）→ /review（週次）
```

---

## 1. `/research` — 今日の有望株を探す

**目的**: 市場全体をスキャンして、今日エントリーすべき候補を5〜8銘柄に絞り込む

### 何をやっているか

| ステップ | 内容 |
|---|---|
| ① 相場環境の確認 | VIX・SPYの状態を見て「今は攻めていい相場か（NORMAL / 暴落中 / 下降トレンド）」を判定 |
| ② 動いている銘柄の取得 | yfinanceから当日の上昇率・出来高急増銘柄を取得。ウォッチリストも優先チェック |
| ③ 候補を5〜8銘柄に絞る | 小〜中型株、出来高1.5倍超、明確な上昇理由があるものを選択 |
| ④ 各銘柄を8種類のデータで調査 | 株価・テクニカル・財務・ニュース・アナリスト評価・決算日・相対強度など |
| ⑤ スコアリング（5軸×重み付け） | モメンタム(25%) / ファンダメンタルズ(20%) / カタリスト(25%) / テクニカル(15%) / センチメント(15%) |

**合格ライン**: 総合スコア **7.0以上**のみ次のステップへ

### 中間成果物

| ファイル | 内容 |
|---|---|
| `data/research_history.json` | 候補銘柄の全データ（スコア・目標株価・損切り・根拠など）をJSON保存。次の`/decision`が読み込む |
| `reports/research/research_YYYY-MM-DD.md` | 人間が読むためのレポート。スコア表・銘柄詳細・エントリーゾーン・リスクなど |

---

## 2. `/decision` — 買うか・見送るかを判定する

**目的**: リサーチ結果に対して、複数の「投資家ペルソナ」が討論し、最終的な売買判断を下す

### 何をやっているか

**5人の架空の投資家キャラクターが審査する**

| ペルソナ | モデルにした人物 | 視点 |
|---|---|---|
| The Oracle | ウォーレン・バフェット | 事業の堀・割安かどうか・キャッシュフロー |
| The Innovator | キャシー・ウッド | 破壊的イノベーション・5年後の市場規模 |
| The Macro Mind | レイ・ダリオ | マクロ経済・金利・相場全体のリスク |
| The Tenbagger | ピーター・リンチ | 成長株の割安感・わかりやすいストーリー |
| The Tape Reader | ジェシー・リバモア | チャート・出来高・モメンタム |

銘柄のセクターや相場環境に応じて3〜4人が自動選出され、以下3ラウンドで争う。

| ラウンド | 内容 |
|---|---|
| ① 独立スタンス | 各ペルソナがデータだけを見てBUY / WAIT / PASSを表明 |
| ② クロスファイア討論 | 意見の異なるペルソナが互いを論破しようとする |
| ③ PM（ポートフォリオマネージャー）最終判断 | 討論結果を踏まえてBUY or PASSを決定。ポジションサイズも算出 |

**ポジションサイズ（ハーフケリー基準）**:

| 確信度 | 投資額 |
|---|---|
| HIGH | ~$3,350 |
| MEDIUM | ~$1,675 |
| LOW | ~$670 |

### 中間成果物

| ファイル / 送信先 | 内容 |
|---|---|
| `data/research_history.json`（追記） | 討論結果（各ペルソナの最終スタンス・PM判断）を同ファイルに保存 |
| `reports/decision/decision_YYYY-MM-DD.md` | 討論サマリー・推奨銘柄・エントリー条件・根拠をまとめたレポート |
| Slack通知 | 推奨銘柄のみ（討論内容は含まない）をSlackに送信 |

---

## 3. `/monitor` — 保有ポジションを毎日チェックする

**目的**: 現在保有している銘柄の損益・アラート状態・テクニカルを毎日確認する

### 何をやっているか

| ステップ | 内容 |
|---|---|
| ① ポジション読み込み | `data/portfolio.csv`から保有中の銘柄を取得 |
| ② 最新株価・テクニカル取得 | RSI / MACD / EMA / ボリンジャーバンドなど |
| ③ 損益計算 | 含み益・含み損、目標株価まで何%残っているか |
| ④ アラート判定 | 6種類のルールで自動判定（下記参照） |
| ⑤ Slack送信 | `scripts/run_monitor.py`経由でSlack通知 |

**アラート条件（ルールベース、AIなし）**:

| 条件 | 深刻度 | 意味 |
|---|---|---|
| 現在値 ≤ 損切りライン | 🔴 HIGH | 損切り発動 → 即対応 |
| 現在値 ≥ 目標株価 | 🔴 HIGH | 目標到達 → 利確検討 |
| 損切りまで3%以内 | 🟡 MEDIUM | 損切り近接注意 |
| 目標まで3%以内 | 🟡 MEDIUM | 目標間近 |
| 含み損 -5%超 | 🟡 MEDIUM | 下落警戒 |
| 含み益 +15%超 | 🔵 INFO | 大幅利益（管理確認） |

### 中間成果物

| ファイル / 送信先 | 内容 |
|---|---|
| `data/monitor_history.json` | 毎回の監視結果を時系列で蓄積（損益・アラート履歴） |
| `reports/monitor/monitor_YYYY-MM-DD.md` | ポジション詳細・アラート一覧・ウォッチリスト値動き・推奨アクションを記載 |
| Slack通知 | HIGH/MEDIUMアラートがある場合にメッセージ送信 |

---

## 4. `/review` — 予測精度を振り返る

**目的**: 過去の投資判断が正しかったか分析し、スコアリング基準を改善する

### 何をやっているか

`data/research_history.json`に蓄積された過去データを使って：

- **スコア別的中率**: スコア8.0以上の銘柄は実際に勝てているか？
- **確信度別の成績**: HIGH確信 vs MEDIUM確信で結果に差があるか？
- **どの評価軸が最も予測精度が高いか**: モメンタム・ファンダメンタルズ・カタリストのどれが実際のリターンと連動するか
- **スコアキャリブレーション提案**: 重みや閾値の見直し提案

### 中間成果物

| ファイル | 内容 |
|---|---|
| Markdownレポート（画面表示） | 分析結果 + キャリブレーション提案 |
| `reports/review/review_YYYY-MM-DD.md` | 上記を保存（任意） |

## データの流れ全体図

```
/research
  ↓ Python(yfinance)でデータ収集
  ↓ Claude がスコアリング・候補選別
  → data/research_history.json に保存
  → reports/research/research_YYYY-MM-DD.md を出力

/decision
  ↓ research_history.json を読み込み
  ↓ ペルソナ討論（3ラウンド）
  → 同 JSON に討論結果を追記
  → reports/decision/decision_YYYY-MM-DD.md を出力
  → Slack に推奨銘柄を送信

  （手動: scripts/add_position.py でポジション追加）

/monitor（毎日実行）
  ↓ portfolio.csv を読み込み
  ↓ 最新データ取得 → 損益・アラート計算
  → data/monitor_history.json に蓄積
  → reports/monitor/monitor_YYYY-MM-DD.md を出力
  → Slack に通知

/review（週次・任意）
  ↓ research_history.json + monitor_history.json を分析
  → スコアリング改善提案をレポートに出力
```

## 主要ファイル一覧

| ファイル | 役割 |
|---|---|
| `data/portfolio.csv` | 保有ポジション（買値・目標・損切りなど） |
| `data/research_history.json` | 全リサーチ結果・討論ログの永続ストア |
| `data/monitor_history.json` | 毎日の監視結果の時系列ログ |
| `data/watchlist.json` | ウォッチリスト銘柄 |
| `data/cache/` | yfinanceデータの24時間キャッシュ |
| `reports/research/` | リサーチレポート（日付別） |
| `reports/decision/` | 投資判断レポート（日付別） |
| `reports/monitor/` | モニターレポート（日付別） |
| `.claude/commands/` | 各コマンドの実行手順定義 |
| `scripts/tool.py` | データ取得の共通CLI（全コマンドが使用） |
