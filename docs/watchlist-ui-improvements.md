# Watchlist UI 改善計画

> 作成日: 2026-04-18  
> 対象ファイル: `skills/dashboard.py` → `render_watchlist()`

---

## 現状の課題整理

現在のWatchlistカードが持つ情報:
- Ticker名 / モニターフラグバッジ / Portfolio入りバッジ
- リサーチスコア (★ X.X)
- 参照価格 / 現在値
- vs参照 騰落率 / 本日騰落率
- reasonテキスト（90文字截断）

### 主要な問題点

| # | 問題 | 影響 |
|---|------|------|
| 1 | ソート・並び替えができない | アラート優先で目視ソートが必要 |
| 2 | エントリーゾーンが見えない | reasonテキストに埋もれていて判断できない |
| 3 | ウォッチ歴（追加日）が見えない | 古い候補と新鮮な候補が区別できない |
| 4 | reasonテキストが90文字で切れる | 重要な条件やカタリストが読めない |
| 5 | スコアが数値だけで直感的でない | 7.9と8.1の差が視覚的にわからない |
| 6 | promoted銘柄がグレーアウトのまま残る | ノイズになる。折りたたむ or 完全分離すべき |
| 7 | 価格のトレンドが見えない（スパークライン無し） | 上昇中か下落中かがわからない |
| 8 | フィルタリングのみでソートがない | 「スコア順」「vs参照 騰落率順」などで見たい |
| 9 | カードサイズが一定で優先度が伝わらない | アラートカードと監視中カードが視覚的に同格 |
| 10 | カード内に「何をするべきか」が書かれていない | リサーチ済み/未リサーチの状態が不明確 |

---

## ベストプラクティス参考（TradingView / Webull / Bloomberg）

- **TradingView**: 任意カラムでのソート、カスタムセクション、サマリー行（平均・最大・最小）、インラインアラート設定
- **Webull**: グループコードで他ウィジェットとリンク、リアルタイム同期、コンパクトな1行表示
- **Bloomberg**: スコアリング列の視覚化、ヒートマップ的な色分け、クイックアクションボタン
- **共通原則**: 情報の密度よりスキャナビリティ優先、アラートは即目につく位置に、カスタマイズ性

## 改善タスク一覧

### 🔴 優先度 HIGH（意思決定への直接影響）

---

#### Task 1: エントリーゾーンの視覚化

**目的**: 「今すぐBUYできるか」を一目で判断できるようにする

**現状**: entryゾーン情報がreasonテキスト内に自然言語で埋もれている  
例: `"エントリーゾーン $116–122、目標 $134.12"`

**改善案**:
- `reason` テキストから正規表現でエントリーゾーン・目標価格を抽出
- カード内に簡易価格バーを追加:
  ```
  [エントリーゾーン]    [現値●]    [目標]
  $116────$122          $141       $134
  ```
- 現値がエントリーゾーン内なら **グリーン強調**、外れていれば通常色

**実装メモ**:
```python
# reasonから抽出する正規表現
import re
entry_match = re.search(r'エントリーゾーン \$([0-9,]+)(?:–\$?([0-9,]+))?', reason)
target_match = re.search(r'目標 \$([0-9,]+)', reason)
```

---

#### Task 2: ソート機能の追加

**目的**: 注目度の高い銘柄を素早く浮上させる

**改善案**: タブの横にソートドロップダウンを追加

| ソートキー | 説明 |
|-----------|------|
| デフォルト | alert優先 → スコア降順 |
| スコア順 | リサーチスコア降順 |
| vs参照 | 参照価格からの騰落率降順（上がった順） |
| 本日騰落 | 当日の動き降順 |
| 追加日 | 最近追加した順 |

**実装メモ**:
- Pythonサーバー側でソートして渡す（JSソートでも可）
- ドロップダウン: `<select id="wl-sort">` + JSでカードを並び替え

---

#### Task 3: アラートカードの視覚的差別化強化

**目的**: アラートカードを「即判断が必要」と一目でわかるように

**現状**: amber borderのみ、カードサイズは同じ

**改善案**:
- アラートカードを **カードグリッドの上に横幅フル** で表示（バナー型）
- 左端に太いamberボーダー（4px）
- 「要アクション」ラベルを右上に常時表示
- アイコン: ⚡ を大きく表示（18px）
- 背景: `#fffbeb` → `#fef9c3`（もう少し目立つ黄色）

**レイアウトイメージ**:
```
┌──────────────────────────────────────────────┐ ← 横幅100%
│ ⚡ CRDO  📈 急騰シグナル        [要アクション] │
│ 参照: $119  現値: $160  +34.4% vs参照         │
│ エントリーゾーン: $116–$122 ──●現値●── $134目標│
└──────────────────────────────────────────────┘
```

---

#### Task 4: reasonの展開表示（アコーディオン）

**目的**: 重要な投資テーマ・条件を全文で読めるようにする

**現状**: 90文字で強制截断。カタリストや条件が読めない

**改善案**:
- 常時表示は40文字（投資テーマ冒頭のみ）
- 「▼ 詳細」クリックでカード内に全文展開（アコーディオン）
- 展開時にはリサーチ日時も表示

**実装メモ**: JSの `details/summary` または classList toggle

### 🟡 優先度 MEDIUM（使いやすさ向上）

---

#### Task 5: スコアバーの視覚化

**目的**: スコアの相対的な高低を直感的に把握する

**現状**: `★ 7.9` のテキスト表示のみ

**改善案**:
```
★ 7.9  [███████░░░] 10点中
```
- プログレスバー形式（0〜10スケール）
- 8.0以上: emerald / 6.0〜8.0: amber / 6.0未満: slate
- 未リサーチ（スコアなし）は「未リサーチ」バッジを表示

---

#### Task 6: 追加日・ウォッチ歴の表示

**目的**: 長期間BUYできていない銘柄を識別する

**現状**: `added_at` フィールドがあるが非表示

**改善案**:
- カード右下に `追加 2026-04-05 (13日前)` を表示
- 30日以上経過した銘柄には `⏰ 長期監視` バッジ
- これにより「ずっと様子見のまま放置している銘柄」を検出できる

---

#### Task 7: 未リサーチ銘柄の明示

**目的**: `/research --seed` すべき候補を一目で識別

**現状**: `last_score: null` の銘柄（AVGO, MU, VRT等）がスコアなしのまま混在

**改善案**:
- スコアなし銘柄には `🔍 未リサーチ` バッジを表示（amber）
- ソート時に「未リサーチ優先」オプションを追加
- カードの opacity は下げない（むしろ研究候補として目立たせる）

---

#### Task 8: promoted 銘柄の分離・折りたたみ

**目的**: ノイズを減らし、active銘柄に集中する

**現状**: promoted銘柄がgrayout 0.45でグリッドに混在

**改善案**:
- Watchlistセクションのデフォルト表示からpromotedを除外
- 「Portfolio入り」タブのみに表示、デフォルト折りたたみ
- もしくは「Portfolio入り」タブ自体を削除し、promoted銘柄はポートフォリオセクションに一本化

## 実装時の注意事項

- `dashboard.py` はデータ取得とHTML生成が分離している（Python → HTML注入）
- CSS/JSは `_CSS` / `_JS` の文字列定数に追記（f-string不可なので `{` はエスケープ不要）
- エントリーゾーン抽出は `reason` フィールドの書式に依存するため、watchlist.json の書式統一が前提
- スパークライン追加時は yfinance への追加リクエストが発生（キャッシュ戦略を確認）
